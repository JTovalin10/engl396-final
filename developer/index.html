Conceptual Overview

How the files in My Favorite Albums work together to produce the application.

The codebase uses R with the Shiny, dplyr, and ggplot2 libraries. If you are familiar with any event-driven web framework — such as Flask, Django, or Express — the structure will feel familiar. The core idea is the same: a UI layer collects user input, a server layer processes it, and helper modules handle the logic for each feature.

The dataset (Data/album-ranking.csv) is the single data source for the entire application. It contains the following fields: Year, Ranking, Album, Artist, Rating, Vinyl, EP, and Live.

Prerequisites: familiarity with general programming concepts. Knowledge of R is helpful but not required.

How the App is Structured

The application is divided into three layers:

  1. Entry point — app.R loads all libraries, sources all helper files, and launches the app by connecting the UI and server.
  2. UI layer — app_ui.R defines the layout and input controls the user sees. It reads the CSV once to populate drop-down menus and sliders, then hands off to the server.
  3. Server layer — app_server.R listens for user input and calls the appropriate helper function. When a user changes a drop-down or slider, Shiny automatically re-runs the relevant block of server code — this is called reactivity and is the key concept to understand in any Shiny app.

Each of the remaining files is a self-contained helper module that handles the data logic for one tab. The server calls these functions and renders their output back to the UI.

File Descriptions

app.R — The entry point of the application. It loads the required libraries (Shiny, dplyr, ggplot2), sources every helper file, and calls shinyApp() to launch the app by connecting app_ui.R and app_server.R. To run the app locally, open this file in RStudio and click Run App, or run shiny::runApp() from the project directory in the R console.

app_ui.R — Defines what the user sees. It loads the album data from the CSV and uses it to populate the controls — drop-down menus, sliders, and checkboxes — across seven tabs. Each tab specifies its inputs and a placeholder for its output. The actual data that fills those placeholders is determined by app_server.R.

app_server.R — The bridge between the UI and the helper files. For each tab, it reads the user's selection and passes it to the corresponding helper function, then renders the result. Because Shiny is reactive, this happens automatically whenever the user changes an input — no manual refresh needed.

home.R — Provides three summary statistics displayed on the home tab: the total number of albums in the dataset, the total number of unique artists, and the artist with the most albums.

number_one_albums.R — Takes a start year and end year from the range slider and returns the top-ranked album for each year in that range, sorted by year in ascending order.

albums_by_year.R — Takes a single year from a drop-down menu and returns a table of all albums ranked that year, showing Ranking, Album, and Artist sorted by ranking order. The all_years variable pre-populates the drop-down by extracting every unique year from the dataset.

albums_by_band.R — Takes an artist name and returns a table of that artist's albums showing Album, Year, and Rating sorted by year. Two helper functions — band_mean_rating() and band_album_count() — compute the average rating and total album count displayed below the table. All three functions share the same filter() and select() pattern from dplyr. The all_bands variable pre-populates the drop-down alphabetically.

fav_bands.R — Takes two inputs: a minimum album count and whether to include EPs and live albums. It filters the dataset accordingly and returns the top 15 artists sorted by average rating in descending order.

compare_bands.R — Takes two artist selections and produces a comparative line chart using ggplot2. Each artist's average rating per year is plotted as a line — red for the first artist, blue for the second.

vinyl.R — Takes a minimum rating and returns a table of top-rated albums the owner does not own on vinyl, sorted by rating descending and then alphabetically by album name.

Data/album-ranking.csv — The CSV file that all data aggregation functions operate on. It is loaded once in app_ui.R to populate the UI controls, and again in each helper file via the shared album_data dataset.

Key Concept: Shiny Reactivity

In a standard script, code runs top to bottom once. In a Shiny app, the server function contains reactive blocks that re-execute automatically when their inputs change. For example, when a user picks a different artist in the Artist's Albums tab, Shiny detects that the input changed and re-runs the block that calls albums_by_bands() — without any additional wiring required. Understanding this eliminates most confusion about why code in app_server.R is organized the way it is.

Setting Up R, RStudio, and Required Packages

Use these steps to install R, RStudio, and the packages required to run My Favorite Albums. The app relies on three packages: shiny to build the web interface, dplyr to filter and aggregate album data, and ggplot2 to generate the artist comparison chart. All three must be installed before the app runs.

  1. Download and install RStudio from the Posit website. Select the free RStudio Desktop version and follow the installer prompts.
  2. Install the required packages:
    1. Open RStudio.
    2. In the console, run: 
      install.packages(c("shiny", "dplyr", "ggplot2"))
    3. Wait for the console prompt (>) to reappear — this indicates installation is complete.
  3. Set your working directory to the project folder via Session > Set Working Directory > Choose Directory.

Setting Up album-ranking.csv

This is the CSV file that all data aggregation functions operate on. Follow these steps to create or update it with your own album data. Getting this file structured correctly is required before the app loads and displays any data.

  1. In Excel or Google Sheets, create a new blank file and enter the following column headers in the first row in this exact order: 
    Year,Ranking,Album,Artist,Rating,Vinyl,EP,Live
  2. Add each album as its own row beneath the headers. For the last three columns:
    • Vinyl: enter v if you own the album on vinyl, leave empty otherwise
    • EP: enter EP if the release is an EP, leave empty otherwise
    • Live: enter Live if the release is a live album, leave empty otherwise
  3. Save the file as a CSV:
    1. Name the file album-rankings.csv.
    2. Place it in the data/ folder inside the My Favorite Albums project folder.
  4. Open RStudio and set your working directory to the project folder via Session > Set Working Directory > Choose Directory.
  5. Run the app with shiny::runApp(). If the Home tab displays a total album count and artist count, the file loaded correctly.

Note: the Artist field must be spelled consistently across all rows. Inconsistent capitalization (e.g., "Radiohead" vs. "radiohead") causes that artist to appear as multiple separate entries throughout the app.

Note: the year range slider is hardcoded to 1993–2024. If your data includes years outside that range, see app_ui.R for instructions on updating it.

app.R — Running the Application

Use these steps to launch the My Favorite Albums Shiny web application. app.R is the entry point that loads all libraries and sources every other R file in the project.

  1. Open app.R in RStudio.
  2. Set your working directory to the project folder via Session > Set Working Directory > Choose Directory.
  3. Click the Run App button in the top-right corner of the editor pane, or run: 
    shiny::runApp()
    The application opens in your browser with all seven tabs available.

Note: add new library() and source() calls to this file when extending the app with additional dependencies or R files.

app_ui.R — Defining the Application Layout

Use these steps to modify the layout or add a new tab. app_ui.R controls everything the user sees — keeping it separate from the server logic makes both files easier to maintain and update independently.

  1. Open app_ui.R in RStudio.
  2. Each tab is defined as a tabPanel() inside tabsetPanel(): 
    tabPanel("Tab Name",
  selectInput("your_input", "Label", choices = c(...)),
  actionButton("your_submit", "Submit"),
  tableOutput("your_output")
)
    • To add a new tab:
      1. Find the last tabPanel() inside tabsetPanel().
      2. Add your new tabPanel() directly after it, before the closing ) of tabsetPanel().
      3. Set the input ID (e.g., "your_input") and submit ID (e.g., "your_submit") — these must match what you use in app_server.R.
      4. Set the output ID (e.g., "your_output") — this must match the ID used in renderTable() or renderPlot() in app_server.R.
      5. Use plotOutput() instead of tableOutput() if your function returns a chart.
    • To modify an existing tab:
      1. Find the tabPanel() by its name (e.g., tabPanel("Vinyl", ...)).
      2. Update the input control choices, labels, or IDs as needed.
      3. If you change an input or output ID, update the matching ID in app_server.R to keep them in sync.
  3. Save the file and re-run the app with shiny::runApp() to see your changes.

Note: the Number One Albums tab uses a range slider with hardcoded bounds (min = 1993, max = 2024). If your data includes years outside this range, update the sliderInput() call inside the tabPanel("Number One Albums", ...) block: 

sliderInput("year_range", "Select Year Range",
            min = 1990, max = 2030,
            value = c(1993, 2024))

app_server.R — Connecting Input to Data Functions

Use these steps to trace or update how a tab's user input connects to a data function. app_server.R handles all reactivity — it listens for user input and decides what to compute and display in response.

  1. Open app_server.R in RStudio.
  2. Locate the tab you want to work with. Each tab calls one or more functions:
    • Home: total_album_count(), total_band_count(), most_pop_artist()
    • Number One Albums: number_one_album()
    • Top Albums by Year: year_albums()
    • Artist's Albums: albums_by_bands(), band_album_count(), band_mean_rating()
    • Favorite Artists: favorite_bands()
    • Artist Comparison: band_album_comparison_chart()
    • Vinyl: missing_vinyl()
  3. Each tab's server logic follows this pattern: 
    observeEvent(input$submit_id, {
  output$output_id <- renderTable({
    function_name(input$input_id)
  })
})
    • To add a new block:
      1. Find the last observeEvent() in the file.
      2. Add a new block directly after it, inside the server <- function(input, output) body.
      3. Set submit_id and output_id to match the IDs you defined in app_ui.R.
      4. Replace function_name with the function you want to call.
    • To modify an existing block:
      1. Find the block by searching for the tab's submit ID (e.g., input$submit_vinyl).
      2. Replace function_name with your new function.
      3. Update the argument to match the relevant input control.
      4. If your function returns a chart, use renderPlot() instead of renderTable().
  4. Save and run the app with shiny::runApp(). Interact with the tab you added or modified and confirm it returns the expected output.

Adding a New Tab and Function

Use these steps to add a custom R file with its own aggregation function and expose it as a new tab. This requires changes to three files: app.R, app_ui.R, and app_server.R.

  1. Write your function in a new R file in the project directory. Your function can reference album_data directly — it is a global variable loaded automatically when the app starts: 
    your_function <- function(your_input) {
  album_data %>%
    filter(Column == your_input) %>%
    select(Album, Artist, Rating) %>%
    arrange(desc(Rating))
}
    Replace Column with the field you want to filter on (e.g., Artist, Year) and update select() to include whichever columns you want returned.
  2. Open app.R. Find the block of existing source() calls and add a new line: 
    source("your_file.R")
  3. Open app_ui.R. Add a new tabPanel() after the last existing one, before the closing ) of tabsetPanel(): 
    tabPanel("Your Tab Name",
  selectInput("your_input", "Label", choices = c(...)),
  actionButton("your_submit", "Submit"),
  tableOutput("your_output")
)
    Choose IDs for "your_input", "your_submit", and "your_output" — you will use these same IDs in the next step.
  4. Open app_server.R. Add a new observeEvent() block after the last existing one, using the same IDs you defined in app_ui.R: 
    observeEvent(input$your_submit, {
  output$your_output <- renderTable({
    your_function(input$your_input)
  })
})
  5. Save all three files and run: 
    shiny::runApp()
    Your new tab appears in the tab bar. Clicking Submit displays the result of your function.

Note: the output ID in tableOutput() must exactly match the ID in renderTable(), and input IDs must match between app_ui.R and app_server.R.

API Reference

Reference documentation for all data aggregation functions. All functions operate on the album_data dataset loaded from data/album-ranking.csv.

Quick Start

Use these functions in the RStudio console. First load the data and required packages, then source the file containing the function you need.

1. Load data and packages

library(dplyr)
album_data <- read.csv("data/album-ranking.csv")

Set your working directory to the project folder first (Session → Set Working Directory → Choose Directory).

2. Source the file and call the function

source("home.R")
total_album_count()
total_band_count()
most_pop_artist()

Output: "Albums in the database: 123", "Artists in the database: 45", etc.

3. Example: Get an artist's albums

source("albums_by_band.R")
albums_by_bands("Radiohead")
band_mean_rating("Radiohead")
band_album_count("Radiohead")

4. Example: Compare two artists (requires ggplot2)

library(ggplot2)
source("compare_bands.R")
band_album_comparison_chart("Radiohead", "Arcade Fire")

Artist names are case-sensitive. Run unique(album_data$Artist) to see valid names.

Ranks top artists by average rating with configurable filters.

Returns up to 15 artists (by average rating) with at least min_albums.var ranked albums. Sorted in descending order by rating. When live_ep.var == TRUE, excludes EPs and live albums.

Parameters
NameTypeRequiredDescription

Returns: data.frame with columns Artist, Rating (average), NumAlbums (sorted descending by rating)

FAQ

Why doesn't my artist name work?

Artist names are case-sensitive and must match exactly. Run unique(album_data$Artist) in R to see all valid names.

How do I load the data?

Set your working directory to the project folder, then run album_data <- read.csv("data/album-ranking.csv"). You need dplyr for most functions. See Quick Start for the full setup.

I get an empty table. What's wrong?

Check that the year or artist exists in your data. Use unique(album_data$Year) and unique(album_data$Artist) to see valid values.

Where is the Shiny app?

Run app.R in RStudio to launch the web interface. Make sure your working directory is the project folder.

Where can I get more technical detail?

The MyFavoriteAlbums source code has detailed comments for each function: behavior, examples, exceptions, returns, and parameter specs.
