My Favorite Albums

API Reference

Reference documentation for the data aggregation functions used by the My Favorite Albums Shiny app. All functions operate on the album_data dataset (loaded from data/album-ranking.csv). Developers who want more detail—including behavior, examples, exceptions, returns, and parameters for each function—should look at the MyFavoriteAlbums repository, where the source code has more detailed comments.

Quick Start

Use these functions in the RStudio console. First load the data and required packages, then source the file containing the function you need.

For developers: The MyFavoriteAlbums GitHub repo has more detailed documentation in the code (behavior, examples, exceptions, returns, parameters).

1. Load data and packages

library(dplyr)
album_data <- read.csv("data/album-ranking.csv")

Set your working directory to the project folder first (Session → Set Working Directory → Choose Directory).

2. Source the file and call the function

source("home.R")
total_album_count()
total_band_count()
most_pop_artist()

Output: "Albums in the database: 123", "Artists in the database: 45", etc.

3. Example: Get an artist's albums

source("albums_by_band.R")
albums_by_bands("Radiohead")
band_mean_rating("Radiohead")
band_album_count("Radiohead")

4. Example: Compare two artists (requires ggplot2)

library(ggplot2)
source("compare_bands.R")
band_album_comparison_chart("Radiohead", "Arcade Fire")

Artist names are case-sensitive. Run unique(album_data$Artist) to see valid names.

number_one_album

Retrieves the top album (Ranking == 1) for each year in a range.

GET number_one_album(var.startyear, var.endyear)

Filters album_data to rows where Year is in range and Ranking == 1. Returns one row per year. Sorted in ascending order by year.

Parameters
NameTypeRequiredDescription
var.startyearintegerYesInclusive start of year range (earliest year to include)
var.endyearintegerYesInclusive end of year range (latest year to include)
Returns: data.frame with columns Year, Album, Artist (sorted in ascending order by year)
Example 
source("number_one_albums.R")
number_one_album(2018, 2024)
#   Year        Album      Artist
# 1 2018  Golden Hour   Kacey Musgraves
# 2 2019  Norman F***ing Rockwell!  Lana Del Rey
# ...

favorite_bands

Ranks top artists by average rating with configurable filters.

GET favorite_bands(min_albums.var, live_ep.var)

Returns top 15 artists (by average rating) with at least min_albums.var ranked albums. Sorted in descending order by rating. When live_ep.var == TRUE, excludes EPs and live albums.

Parameters
NameTypeRequiredDescription
min_albums.varintegerYesMinimum number of ranked albums an artist must have to appear in results
live_ep.varlogicalYesIf TRUE, excludes EPs and live albums from the calculation; if FALSE, includes all album types
Returns: data.frame with columns Artist, Rating (average), NumAlbums (sorted in descending order by rating)
Example 
source("fav_bands.R")
favorite_bands(3, TRUE)   # Top 15 artists, min 3 albums, exclude EPs/live
favorite_bands(5, FALSE)  # Include all album types
#   Artist Rating NumAlbums
# 1 Radiohead  8.9        12
# 2 Arcade Fire  8.7         6
# ...

band_album_comparison_chart

Generates a line chart comparing two artists' ratings by year.

GET band_album_comparison_chart(var.artist1, var.artist2)

Plots each artist's album ratings by year. First artist = red line, second = blue. Requires ggplot2.

Parameters
NameTypeRequiredDescription
var.artist1characterYesFirst artist (plotted as red line on chart)
var.artist2characterYesSecond artist (plotted as blue line on chart)
Returns: ggplot object (displays in RStudio Plots pane)
Example 
library(ggplot2)
source("compare_bands.R")
band_album_comparison_chart("Radiohead", "Arcade Fire")

Renders a line chart in the Plots pane. Red = first artist, blue = second. X-axis: year, Y-axis: rating.

missing_vinyl

Lists highly rated albums not yet owned on vinyl.

GET missing_vinyl(missing_vinyl_rating.var)

Filters album_data to rows where Rating ≥ threshold and Vinyl is empty. Returns album, artist, rating. Sorted in descending order by rating.

Parameters
NameTypeRequiredDescription
missing_vinyl_rating.varnumericYesMinimum rating threshold (albums with rating ≥ this value are included)
Returns: data.frame with columns Album, Artist, Rating (sorted in descending order by rating)
Example 
source("vinyl.R")
missing_vinyl(8.5)
#              Album        Artist Rating
# 1  To Pimp a Butterfly  Kendrick Lamar    9.5
# 2  Blonde            Frank Ocean      9.2
# ...

Data Schema

Structure of album-ranking.csv used by all functions.

FieldTypeDescription
YearintegerRelease year
RankingintegerOwner's rank for that year (1 = top pick, 2 = second, etc.)
AlbumcharacterAlbum title
ArtistcharacterArtist name (must be consistent)
RatingnumericOwner's rating (typically 0–10 scale)
VinylcharacterEmpty = not owned; non-empty = owned
EPcharacter"EP" for extended play; otherwise excluded
Livecharacter"Live" for live albums; otherwise excluded

Artist names are case-sensitive. Use unique(album_data$Artist) for valid values. See also all_bands and all_years variables for populating UI dropdowns.

Example: List valid artist names 
unique(album_data$Artist)
# [1] "Radiohead"   "Arcade Fire" "Taylor Swift" ...

FAQ

Common questions for end users and developers.

Why doesn't my artist name work?

Artist names are case-sensitive and must match exactly what is in the data. "Radiohead" is not the same as "radiohead". Run unique(album_data$Artist) in R to see all valid names.

How do I load the data?

Set your working directory to the project folder, then run album_data <- read.csv("data/album-ranking.csv"). You need dplyr for most functions. See Quick Start for the full setup.

I get an empty table. What's wrong?

Check that the year or artist exists in your data. For year_albums, the year must be in album_data$Year. For artist functions, the name must match album_data$Artist exactly. Use unique(album_data$Year) and unique(album_data$Artist) to see valid values.

Where is the Shiny app?

Run app.R in RStudio to launch the web interface. The app loads the data and all function files automatically. Make sure your working directory is the project folder.

Where can I get more technical detail?

The MyFavoriteAlbums source code has detailed comments for each function: behavior, examples, exceptions, returns, and parameter specs with input expectations.
